Implementação de Árvore-B

O nó contém uma matriz de estruturas-base, contendo a chave e dois
ponteiros, um para elementos menores e outro para elementos maiores que
aquele.

Essa implementação não é razoável, pois insere complexidade não
necessária. Com algumas orientações do professor usarei C++ e todas as
operações nos nós serão encapsuladas numa classe, o que simplifica
bastante. Também com as orientações do professor essa classe pode ter

int	howmany; // how many keys in the node
key	keys[MAXAGESIZE-1]; // the keys in the page/node
key*	pointers[MAXPAGESIZE]; // the pointers to other pages

Bem, como o nosso elemento é bem simples (o bom e velho tipo_elem), não
há problema em ter um vetor com elementos de fato, pois quando for parar
na memória (considerando o usual caso de acesso a discos), vai com todo
mundo daquele bloco/página junto. Mas isso está relacionado com o uso da
B Tree em questão. Adotei o método mais interessante, que é ter apenas a
chave e um ponteiro pra estrutura de verdade, esteja onde estiver:

class ElemKey {
	...

	int key;
	Elem *data;

	...
};
